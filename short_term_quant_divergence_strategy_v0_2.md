# 短线强势股「好图形 + 量化活跃 + 分歧承接」预案系统 v0.2

> 用途：交给 Codex / 自动化代码系统进行数据处理、信号生成、回测实验与后续迭代。  
> 当前状态：研究型策略说明，不是最终交易系统。  
> 核心思想：**只做右侧好图形里的核心分歧；量化活跃负责入池，承接负责执行，图形趋势负责交易权限。**

---

## 0. 策略一句话定义

本系统用于识别 A 股短线强势票中，**图形趋势仍然好看、疑似有活跃资金/量化资金参与、断板或分歧后存在有效承接**的标的，并在前一日收盘后生成次日交易预案。

最简规则：

```text
好图形里，核心分歧确认，次日关键位低吸验证，修复后按短线纪律兑现。
```

不做：

```text
破位丑图里的反抽承接；
高位巨量失败后的弱修复；
没有题材增量、只有防守承接的后排票；
尾盘偷修复但次日无确认的票。
```

## 2. 系统总框架

系统不是盘中临时追涨系统，而是 **盘后预案系统**。

基础时序：

```text
D0：涨停 / 强势上涨 / 进入雷达池
D1：断板或大分歧，观察是否有活跃资金与承接
D1 收盘后：生成 D2 交易预案
D2：只验证预案，不临时选股
D3-Dn：修复、涨停、冲高、失败确认、止盈止损
```

核心判断顺序：

```text
1. 图形趋势是否好看？
2. 是否近期强势涨停结构？
3. 是否有疑似量化/活跃资金参与？
4. 分歧后是否有承接？
5. 题材环境是否支持继续修复？
6. 次日是否给出关键位低吸验证？
7. 修复或涨停后如何兑现？
```

## 3. 最重要的系统原则

### 3.1 人工最佳买点 ≠ 系统推荐买点

以 605589 为例：

```text
人工最佳点：首分歧当天盘中承接点。
系统推荐点：首分歧收盘后确认，次日执行低吸预案。
```

原因：系统必须避免未来函数。D1 日内发生的分歧、承接、成交、收盘状态，在 D0 收盘后无法确认。因此：

```text
D1 = 确认日
D2 = 执行日
```

### 3.2 图形趋势是前置门槛

系统不应先找“量化做”或“收开锚定”，而应先判断：

```text
这只票是不是仍处在资金愿意继续做的右侧趋势结构中？
```

若图形趋势不合格，即使有活跃资金和分歧承接，也只进入观察池，不进入交易池。

### 3.3 量化活跃是入池条件，不是买入条件

“疑似量化做”只能说明有资金、有弹性、有波动。真正能否交易，要看次日关键位是否有有效承接。

```text
量化活跃：宽筛
分歧承接：严判
图形趋势：权限门槛
```

### 3.4 承接要发生在好图形里才有价值

```text
好图形里的承接 = 机会
丑图里的承接 = 多数只是防守或反抽
高位巨量失败后的承接 = 需要极强修复确认，否则视为风险
```

## 4. 图形趋势评分 graph_quality_score

图形趋势分是系统第一层，建议权重最高。

建议初始权重：

```text
综合预案分 = 图形趋势分 × 35%
           + 量化活跃分 × 25%
           + 分歧承接分 × 25%
           + 题材环境分 × 15%
```

### 4.1 右侧结构

判断是否仍然是右侧强势结构。

加分条件：

```text
近 5-10 日高点抬高；
近 5-10 日低点抬高；
收盘重心上移；
分歧后仍处在阶段高位；
近期不断刷新空间；
没有连续失败后的破位结构。
```

扣分条件：

```text
高点降低；
低点降低；
收盘重心下移；
前一轮已经失败；
涨停只是破位后的反抽；
反抽后无法继续新高。
```

### 4.2 均线结构

五日线只是子项，不能单独决定图形好坏。

重点看：

```text
5 日线方向；
10 日线方向；
20 日线方向；
是否 5 日线 > 10 日线 > 20 日线；
短暂破 5 日线后能否快速修复；
是否连续多日压在 5 日线下方；
5 日线是支撑还是反压。
```

分类：

```text
主升支撑型五日线：可作为低吸参考。
破位修复型五日线：谨慎，不等同支撑。
失效反压型五日线：原则上不做。
```

### 4.3 K 线干净度

加分条件：

```text
涨停后首次或核心分歧；
分歧日不收崩；
没有连续长上影；
没有连续冲高失败；
没有多次破位再修复；
大阴后能快速修复。
```

扣分条件：

```text
连续长上影；
连续冲高回落；
涨停次日无溢价；
多次封板失败；
分歧日收盘接近最低；
高位巨量滞涨。
```

### 4.4 涨停位置

系统需要区分：

```text
主升途中的涨停；
突破平台的涨停；
分歧后修复涨停；
破位后的反抽涨停；
高位末端加速涨停。
```

优先：主升途中、突破平台、分歧后强修复涨停。  
谨慎：破位后反抽涨停、高位末端加速涨停。

### 4.5 压力结构

需要识别上方是否存在明显压力：

```text
近期失败高点；
长上影密集区；
高位巨量成交区；
涨停失败价；
A 杀前的出货区。
```

若当前价格接近失败区，但没有强势突破确认，降级。

## 5. 疑似量化/活跃资金评分 active_money_score

系统不直接判断“是否真有量化”，只判断 **疑似活跃资金参与**。

### 5.1 典型特征

```text
当日成交额显著放大；
振幅明显放大；
分时有急拉急杀；
价格围绕开盘价、VWAP、前涨停价、首分歧低点等关键位反复争夺；
不是死水，也不是单边阴跌；
有日内修复尝试；
流动性足够；
同题材或同风格个股有联动。
```

### 5.2 注意事项

收盘价靠近开盘价只是量化活跃的一种表现，不是必要条件。

```text
收开锚定型：大振幅后收盘回到开盘价附近。
日内修复型：早盘分歧后全天慢慢修复。
低开急杀修复型：低开杀到关键位后修复。
尾盘抢修复型：需要谨慎，可能透支次日买盘。
```

## 6. 分歧承接评分 support_score

承接必须分级，不能简单用“不破低点”判断。

### 6.1 A 类：主动修复型承接

优先做。

特征：

```text
回踩关键位不破；
破位后快速收回；
站回 VWAP；
站回开盘价或 D1 收盘价；
收盘位置不低；
次日有修复或溢价；
修复过程不是尾盘偷拉。
```

### 6.2 B 类：防守型承接

只能小仓或观察。

特征：

```text
关键位没破；
但全天压在 VWAP 下方；
没有主动进攻；
收盘位置一般；
题材增量不足；
隔日修复不确定。
```

000700 第一段接近此类。

### 6.3 C 类：无效承接

直接剔除。

特征：

```text
跌破关键位；
放量下跌；
无法收回 VWAP；
收盘接近低位；
隔日继续低开；
题材核心负反馈。
```

## 7. 关键成本区 key_zones

系统执行低吸时不应只看均线，应围绕短线资金成本区。

关键位包括：

```text
首分歧日低点；
首分歧日 VWAP；
前涨停价；
前一日低点；
前一日开盘价；
前一日 VWAP；
涨停启动日成本区；
连续分歧低点区；
失败涨停价；
近期高位成交密集区。
```

判断逻辑：

```text
回踩关键位不破 = 观察承接；
跌破关键位后快速收回 = 强修复信号；
跌破关键位后长时间无法收回 = 预案失效；
高位巨量失败后关键位被跌破 = 趋势破坏风险。
```

## 8. 题材环境 theme_score

题材环境不是主模型，但影响仓位和容错率。

分类：

```text
主线题材：可容忍一到两天分歧，只做核心和强趋势票。
次主线：只做辨识度核心，小仓。
支线轮动：只做强承接，不做弱承接。
孤立个股：原则上不做，除非图形和资金极强。
```

需要记录：

```text
题材涨停家数；
题材内涨幅 > 5% 家数；
题材核心票表现；
龙头是否继续晋级；
后排是否批量负反馈；
市场短线情绪是否退潮。
```

## 9. 主要策略模型

### 9.1 首分歧确认模型

代表：605589。

结构：

```text
连续强板 / 强势涨停
→ D1 首次巨量分歧
→ D1 不收崩，有承接
→ D1 收盘后生成 D2 预案
→ D2 回踩 D1 低点或关键成本区不破
→ 后续修复涨停
```

系统规则：

```text
D1 只确认，不正式追买；
D2 才执行；
D2 不追高，只低吸关键位；
修复涨停后，次日若不能强更强，优先兑现。
```

### 9.2 强趋势多次分歧续强模型

代表：000636。

结构：

```text
高点持续抬高；
低点持续抬高；
均线多头；
多次涨停/断板/分歧；
每次分歧后仍能快速修复或继续新高。
```

系统规则：

```text
不只做首分歧；
只要趋势完整，后续核心分歧也可生成预案；
位置越高，要求越严格；
后期只做强修复，不做弱承接。
```

### 9.3 丑图反抽失败模型

代表：000700 第二段。

结构：

```text
前一波已经失败；
连续破坏趋势；
单日涨停只是反抽；
分歧后看似稳住；
但图形没有重新形成右侧；
后续继续走弱。
```

系统规则：

```text
破位反抽涨停不能直接按首分歧模型处理；
必须重新形成右侧趋势后才考虑；
图形不干净，即使有承接也不做。
```

### 9.4 强趋势末端巨量失败模型

代表：002929。

结构：

```text
前期涨幅巨大；
高位加速；
涨停开盘或冲板；
巨量炸板；
收盘接近全天最低；
收盘显著低于 VWAP；
次日尝试修复但封不住；
后续高位巨量滞涨；
跌破修复区后 A 杀。
```

系统规则：

```text
高位巨量失败日后，只有强修复才能继续做；
弱修复、封板失败、巨量滞涨，都不能买；
后续冲高优先看作卖点；
只有重新涨停突破并站稳失败高点，才重新评估。
```

## 10. 硬过滤规则 hard_filters

初版硬过滤：

```text
1. 图形趋势分过低，不进入交易池。
2. 破位后的单日反抽涨停，不按首分歧模型处理。
3. 高位巨量炸板且收盘接近最低，次日不能强修复则降级为风险票。
4. 分歧日收盘接近最低且显著低于 VWAP，谨慎或剔除。
5. D2 跌破关键位后无法快速收回，预案失效。
6. 只守住低点但不能站回 VWAP，定义为弱承接，不能重仓。
7. 尾盘抢修复但次日无确认，不追。
8. 非主线题材中的弱承接，只观察或小仓。
9. 连续长上影、连续冲高失败、连续封板失败，降低图形分。
10. 修复涨停后次日不能继续强更强，默认兑现或降仓。
```

## 11. 执行规则 execution_rules

### 11.1 D1 收盘后生成预案

输入：D1 日线 + D1 5 分钟数据。

输出：

```text
信号类型；
图形趋势分；
活跃资金分；
承接分；
题材分；
关键成本区；
次日低吸区；
次日失效位；
是否允许参与；
建议仓位等级。
```

### 11.2 D2 执行条件

允许执行：

```text
回踩关键位不破；
跌破关键位后 15-30 分钟内收回；
重新站回 VWAP；
重新站回开盘价或 D1 收盘价；
题材核心没有集体负反馈；
不是单边放量下杀。
```

不执行：

```text
高开后一路兑现且无法站回 VWAP；
跌破关键位后无修复；
低开过多且无承接；
题材前排核按钮；
图形已经进入丑图反抽结构；
高位巨量失败后只有弱修复。
```

### 11.3 退出规则

```text
修复涨停后，次日不能继续强更强，优先兑现；
冲高无板，分批止盈；
跌破关键成本区且无法收回，止损；
D2 低吸后当日收盘低于 VWAP 且无修复，次日优先处理；
高位巨量失败后，后续冲高优先视为卖点。
```

## 17. v0.2 核心结论

当前系统核心不再是简单的“涨停断板低吸”，而是：

```text
在右侧好图形中，识别活跃资金参与的核心分歧；
分歧后若关键成本区承接有效，则在次日生成低吸预案；
若图形破坏、题材无增量或高位巨量失败，则即使有承接也不参与。
```

最终压缩为：

```text
趋势完整，分歧才是机会；
趋势破坏，承接多是反抽；
高位巨量失败，弱修复不是买点。
```
